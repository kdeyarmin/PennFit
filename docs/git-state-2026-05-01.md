# Git state inventory — 2026-05-01

## Why this document exists

The Replit workspace's Git UI started showing `UNKNOWN_REF: Remote ref of current branch is missing`. Investigation revealed the workspace has been operating in a **parallel universe** from the GitHub remote (`https://github.com/kdeyarmin/PennFit`) for an extended period. There are now **four divergent git states** and **no automated reconciliation is safe** without human product/engineering decisions. This document captures the full picture so a human can decide.

> **TL;DR**
> - `pennfit.replit.app` is currently serving code from the **Replit workspace**, not GitHub `main`. The two have been drifting for weeks.
> - GitHub `main` has ~100 customer-facing features (subscription health dashboard, communication preferences/DND, comfort guarantee, travel-mode pause, PWA install, canned-reply macros, etc.) that **never reached the workspace** and therefore **are not in production**.

> - This is not a 5-minute `git pull`. It needs a human-led merge plan.

4. **Reset local `main` to track GitHub `main`** once snapshots are pushed:
   ```
   git fetch subrepl-3ppc2e03
   git checkout main
   ```

5. **Re-publish from the new `main`.** This will roll the deployed `pennfit.replit.app` forward by 123 commits. **Do this in a maintenance window** — you're shipping months of unreviewed feature work to production in one cut.

Pick one (recommend: GitHub `main` is canonical, the workspace should always be `git pull`-ed from `subrepl-3ppc2e03/main` before any agent task starts) and document it in `replit.md`.
