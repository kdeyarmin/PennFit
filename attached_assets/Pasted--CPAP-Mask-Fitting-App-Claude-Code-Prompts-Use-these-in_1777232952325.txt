# CPAP Mask Fitting App — Claude Code Prompts

Use these in sequence. Run each, verify the output, then move to the next.

The architecture they collectively build:

- **Frontend:** React + TypeScript + Vite + Tailwind PWA
- **On-device image processing:** MediaPipe Face Mesh in the browser — images never leave the device
- **Backend:** stateless Node/Express recommendation service that only receives numeric measurements and questionnaire answers
- **Hosting target:** HIPAA-compliant infrastructure (AWS with BAA executed)
- **Zero PHI on the server, ever.**

-----

## Context block — paste at the start of every Claude Code session

```
Project context: I'm building a CPAP mask recommendation web app for my DME company. 
Architecture constraints (do not violate these without flagging):

- Stack: React + TypeScript + Vite + Tailwind (frontend), Node + Express + TypeScript (backend)
- All facial image processing happens ON-DEVICE in the browser using MediaPipe Face Mesh.
  Images NEVER leave the user's device. Only numeric measurements are transmitted.
- Backend is fully stateless. No database, no logging of request bodies, no image handling.
- Deployment target is HIPAA-compliant infra (AWS with BAA), but the architecture is designed
  to minimize PHI exposure regardless.
- Quality bar: ESLint strict, TypeScript strict mode, ≥80% test coverage on business logic.
- Work in small commits. Write tests alongside code, not after. Run lint + typecheck + tests
  before declaring any task complete.
```

## Prompt 3 — Face mesh and measurement

```
Add @mediapipe/tasks-vision and build the measurement module in src/features/measure/:

- useFaceMesh hook: accepts a canvas, returns the 478 face landmarks
- extractMeasurements(landmarks, mmPerPixel): pure function returning
  { noseWidth, noseHeight, noseToChin, mouthWidth, faceWidthAtCheekbones } in mm
- calibrateFromCreditCard(canvas): detects the rectangular card edges in the lower frame 
  and computes mm/pixel from ISO/IEC 7810 ID-1 dimensions (85.60 × 53.98 mm)
- calibrateFromIris(landmarks): fallback using average iris diameter 11.7 mm
- qualityCheck(landmarks, frame): rejects frames with face not centered, head tilt >15°, 
  poor lighting, or missing calibration reference. Returns { ok: boolean, reasons: string[] }
- Comprehensive unit tests using fixture landmark JSON files

Document the mm/pixel calibration math in code comments — future-me needs to understand it.
```

## Prompt 4 — Questionnaire engine

```
In src/features/questionnaire/:

- Typed schema for questions: { id, prompt, type, options, weight, affects: MaskTypeWeights }
- src/data/questions.json with the full clinical questionnaire covering: claustrophobia, 
  mouth breathing, sleep position, facial hair, glasses use, frequent congestion, prior mask

Reference clinical guidance: full-face for mouth breathers, nasal pillows for claustrophobic users 
and side sleepers, full-face contraindicated for heavy beard, etc. Cite sources in comments.
```

## How to use these

- Run prompts one at a time. After each, ask Claude Code to summarize what changed and
  run the verification commands. Don’t move on until the previous step is green.
- Repeat the **context block** at the top of any new Claude Code session so the architecture
  constraints stay in scope.
- For large changes within a prompt, ask Claude Code to plan first, then implement in chunks.

## What these prompts do NOT cover

- **Native iOS app with TrueDepth.** Web/MediaPipe is good enough to start. If you later want
  millimeter-grade accuracy, build a native iOS companion using ARKit’s face tracking — that’s
  a separate project.
- **Legal review.** The consent flow has [ATTORNEY REVIEW] placeholders. Do not launch to
  real patients without a healthcare attorney signing off.
- **FDA / SaMD analysis.** A regulatory consultant should opine on whether a mask
  recommendation tool falls under FDA Software-as-a-Medical-Device jurisdiction. Budget for
  this conversation before you launch.
- **Mask catalog.** The seed data uses public product names for testing. You’ll need to replace
  it with your actual inventory, fit data from manufacturer spec sheets, and your pricing.
